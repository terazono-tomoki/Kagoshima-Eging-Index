# Kagoshima-Eging-Index

鹿児島県内の漁港・ポイントに絞ったエギング専用のコンディション判定ツールです。

## 使用している外部 API（データソース）

本アプリは API キー不要 の [Open-Meteo](https://open-meteo.com/) を HTTP で取得しています（`urllib.request`）。

| エンドポイント | 役割 | 取得する変数（hourly） |
|----------------|------|-------------------------|
| `https://api.open-meteo.com/v1/forecast` | 一般気象の予報 | `wind_speed_10m`, `pressure_msl` |
| `https://marine-api.open-meteo.com/v1/marine` | 海洋の予報 | `wave_height`, `sea_surface_temperature`, `sea_level_height_msl` |

### リクエストの共通パラメータ

- 座標: 各地点の緯度・経度（アプリ内 `locations` に固定）
- タイムゾーン: `Asia/Tokyo`
- 週間予報: `forecast_days=7` を指定し、時系列を日ごとに集約
- 単日スナップショット（釣果ログ登録時の気象記録用）: `start_date` / `end_date` を同一日に指定

取得結果は気象・海洋の hourly を時刻で結合し、欠損行を除いたうえで日別に集約します。  
**風・波・水温・気圧**は同日の平均、**潮**は同日の hourly `sea_level_height_msl` の最大値と最小値の差（その日の潮差の目安）からスコア化します。キャッシュは `st.cache_data(ttl=3600)` で約1時間です。

`sea_level_height_msl` は Open-Meteo が提供する **潮汐を含む海面高度のグリッドモデル**です（無料・APIキー不要）。気象庁の潮汐表や港湾ごとの天文潮とは基準・精度が異なり、航海用には推奨されません。本アプリでは「その地点付近でその日の潮の変動が大きいか」の目安に使っています。

## 独自ロジックの概要

「エギング向けの釣りやすさ」は 実験的なスコアリング であり、実海況・潮流・地形は反映していません。参考用途としてください。

### 1. 潮（海面高度モデルからの日次潮差）

Open-Meteo Marine の `sea_level_height_msl` を1時間ごとに取得し、その日の **潮差**（`max(海面高度) − min(海面高度)`）を求めます。潮差が大きいほど「大潮に近い動き」とみなしてスコアを高くします（`tide_score_from_tide_range`）。  
区分ラベル（大潮寄り／中潮寄り／小潮寄り／長潮・若潮寄り）は、潮差を正規化した値の閾値で付けています。

※ これは **気象庁の天文潮（調和潮汐）そのものではなく**、全球モデル上の海面高度です。海岸形状の影響で実潮とずれる場合があります。

### 2. 総合スコア（Open-Meteo の日次平均から）

各要素を 0〜100 に正規化したうえで 重み付き平均 しています（`evaluate_eging_condition`）。

| 要素 | スコアの考え方（概要） | 総合への重み |
|------|-------------------------|--------------|
| 潮 | 日次潮差（海面高度モデル）からの潮スコア | 34% |
| 風 | 弱いほど高評価（風速 × 係数で減点） | 26% |
| 波 | 低いほど高評価（波高 × 係数で減点） | 18% |
| 水温 | 20℃ 付近を最高とし、乖離で減点 | 14% |
| 気圧 | 1016 hPa 付近を目安に安定寄りを評価 | 8% |

総合スコアから **ランク S/A/B/C/D** に変換（閾値は `get_rank`）。

### 3. 釣果ログ連動（任意）

同一ポイントの過去ログが **2件以上** あるとき、当日の気象スナップショットと各ログに保存した気象の **距離** を計算し、類似度スコアから「実績一致: 高／中／低」を表示します（`evaluate_from_catch_records`）。  
距離は風・波・水温・気圧の差に **それぞれ係数** を掛けて足し合わせたもので、線形なヒューリスティックです。

### 4. ローカル保存・認証

- 釣果は `catch_records.json`、写真は `catch_images/` に保存
- 記録欄の閲覧・保存は `.streamlit/secrets.toml` の `records_section_password` で制限可能
